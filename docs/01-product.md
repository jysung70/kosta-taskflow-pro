# 01 — Product

## 목표

> **"지금 누가 뭐 해?"가 사라지게 한다.**

팀 업무를 실시간으로 시각화해 불필요한 구두 확인, 슬랙 ping, 회의를 줄인다.
누구나 한 화면에서 팀 전체의 업무 상태를 파악할 수 있는 상태를 만드는 것이 이 제품의 존재 이유다.

---

## 페르소나

| 항목 | 내용 |
|---|---|
| 대상 | 10명 규모 스타트업의 팀 리더 |
| 연령대 | 30~40대 |
| 상황 | 매일 아침 팀원 개개인에게 진행 상황을 물어봐야 하는 비효율을 겪고 있다 |
| 필요 | 별도 교육 없이 바로 쓸 수 있는 단순한 업무 현황판 |
| 기기 | 사무실 데스크톱 + 이동 중 모바일 |

---

## MVP 범위

### CRUD 4종 — 화면에서 모두 동작

| 동작 | 설명 |
|---|---|
| 추가 (Create) | 업무 제목, 상태, 마감 시각을 입력해 생성 |
| 목록 (Read) | 전체 업무를 상태별로 필터링해 목록 표시 |
| 수정 (Update) | 목록에서 선택 후 제목·상태·마감 시각 인라인 수정 |
| 삭제 (Delete) | 단건 삭제, 확인 다이얼로그 표시 후 처리 |

### 상태 분류

업무는 다음 3가지 상태 중 하나를 가진다.

```
todo  →  in_progress  →  done
```

### 마감 시각 (due_at)

- 날짜 + 시간을 함께 지정하고 표시한다.
- 형식: `YYYY-MM-DD HH:mm` (예: `2026-05-12 18:00`)
- 마감이 지난 업무는 시각적으로 구별 표시한다.

### 라이트 / 다크 테마 토글

- 헤더에 토글 버튼을 배치한다.
- 선택 값은 `localStorage`에 저장해 새로고침 후에도 유지한다.

### 모바일 반응형

- 최소 지원 너비: **360px**
- 레이아웃은 단일 컬럼으로 전환되며 터치 탭 영역을 충분히 확보한다.

## UI 톤 — macOS 스타일

| 요소 | 규칙 |
|---|---|
| 모서리 | 둥근 모서리 (`border-radius: 12px` 기준) |
| 그림자 | 부드러운 다단계 그림자 (harsh drop shadow 금지) |
| 카드 | 반투명 배경 (`backdrop-filter: blur`) |
| 폰트 | 시스템 폰트 스택 (`-apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif`) |
| 색상 | 채도 낮은 중성 팔레트, 포인트 컬러 1개 |
| 애니메이션 | 전환 시 `transition: 150~200ms ease` 적용 |

## 확장 계획 (MVP 이후)

MVP 완료 후 우선순위 순서로 검토한다. MVP 단계에서는 코드에 포함하지 않는다.

| 기능 | 설명 |
|---|---|
| JWT 로그인 | 이메일 + 비밀번호 인증, 세션 관리 |
| 팀 | 멤버 초대, 권한 분리 (리더 / 멤버) |
| Kanban | 드래그&드롭 칸반 보드 뷰 |
| 채팅 | 업무 단위 코멘트 / 팀 채널 |
| CI/CD | GitHub Actions 기반 자동 배포 파이프라인 |

---

## 범위 외 (Out of Scope)

MVP와 확장 계획 모두에서 다루지 않는다.

- 외부 결제 (PG 연동)
- 네이티브 앱 (iOS / Android)
- WebRTC (화상 / 음성)
- 외부 캘린더 연동 (Google Calendar 등)
- 파일 업로드
- WebSocket — MVP는 **폴링**으로 구현하며, 실시간 확장은 이후 단계에서 별도 결정

## 성공 기준

MVP가 다음 조건을 모두 충족하면 완료로 간주한다.

| 기준 | 조건 | 검증 결과 |
|---|---|---|
| 상태 유지 | 새로고침 후 데이터와 테마가 그대로 유지된다 | ✅ |
| 반응형 | 360px 너비에서 레이아웃이 깨지지 않는다 | ✅ |
| 성능 | CRUD API 응답 시간 200ms 이하 (로컬 기준) | ✅ |
| 기능 | CRUD 4종이 화면에서 모두 정상 동작한다 | ✅ |
| 테마 | 라이트 / 다크 토글이 즉시 전환되고 새로고침 후에도 유지된다 | ✅ |

## 성공 기준 검증 결과 — 2026-05-14

### (1) 상태 유지 ✅

- SQLite 영속화로 서버 재시작 후에도 데이터 유지
- 브라우저 새로고침 시 `fetch` 호출로 목록 즉시 복원
- 테마: `localStorage` 저장값 → 페이지 로드 시 FOUC 없이 즉시 적용 확인

### (2) 360px 반응형 ✅

- Playwright MCP DevTools 360px 뷰포트 직접 측정
- 가로 스크롤 없음 확인
- 요소 겹침 없음 확인 (헤더·폼·카드·버튼 모두 정상 배치)

### (3) API 성능 p95 ≤ 200ms ✅

| 항목 | 수치 |
|---|---|
| 측정 횟수 | 50회 |
| p50 (중앙값) | 42ms |
| p95 | 138ms |
| 기준 | 200ms |
| 판정 | **통과** |

전체 엔드포인트 (POST·GET목록·GET단건·PUT·DELETE) 모두 기준 충족
