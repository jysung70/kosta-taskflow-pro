## 절대 규칙

### 1. 추측 금지
불확실한 요구사항은 절대 추측하지 않는다. 모호한 부분은 반드시 사용자에게 확인 후 진행한다.

### 2. 돌발 의존성 금지
새로운 라이브러리·패키지를 추가하기 전에 반드시 사용자에게 승인을 받는다. 기존 스택으로 해결할 수 없을 때만 제안한다.

### 3. 테스트 없이 완료 금지
구현한 기능은 반드시 테스트(단위·통합·E2E 중 적절한 것)를 작성하고 통과한 뒤 완료로 간주한다.

### 4. 시크릿 하드코딩 금지
API 키, 비밀번호, 토큰 등 민감 정보는 절대 코드에 직접 작성하지 않는다. 반드시 환경 변수(`.env`)를 사용한다.

### 5. 폴더 구조 임의 변경 금지
`docs/04-folder-structure.md`에 정의된 폴더 구조를 임의로 변경하지 않는다. 구조 변경이 필요한 경우 먼저 사용자에게 제안하고 승인을 받는다.

## 모호한 요청 처리

요청이 모호하거나 해석이 여러 가지인 경우, **추측하지 않고** 아래 형식으로 선택지 2~3개를 제시한 뒤 사용자의 선택을 기다린다.

```
요청이 모호합니다. 어떤 방향으로 진행할까요?

1. [선택지 A] — 간략한 설명
2. [선택지 B] — 간략한 설명
3. [선택지 C] — 간략한 설명 (선택적)
```
